# JobOS Browser Extension

Chrome Manifest V3 extension for JobOS.

## Features

- **Clerk sign-in** — syncs with the web app via `syncHost` (no manual token copy)
- **Job capture** — LinkedIn, Indeed, and generic job pages
- **AI job assist** — resume fit score, tailoring suggestions, focus topics, cover letter, networking email, interview questions
- **Auto-fill (experimental)** — basic field matching on simple application forms (many ATS sites block this)

## Setup

1. Configure root `.env` and sync app env files:

```bash
pnpm env:setup
```

2. Build or watch the extension:

```bash
pnpm dev:extension
```

3. Load `apps/browser-extension/dist` in `chrome://extensions` (Developer mode → Load unpacked).

4. **Allow the extension in Clerk** (one-time):
   - Copy your extension ID from `chrome://extensions`
   - Clerk Dashboard → **Configure** → **Developers** → **Paths** (or **Settings** → look for **Allowed origins** / **Authorized parties**)
   - Add `chrome-extension://YOUR_EXTENSION_ID`
   - If you cannot find the UI, use the Clerk Backend API:
     ```bash
     curl -X PATCH "https://api.clerk.com/v1/instance" \
       -H "Authorization: Bearer YOUR_CLERK_SECRET_KEY" \
       -H "Content-Type: application/json" \
       -d "{\"allowed_origins\":[\"chrome-extension://YOUR_EXTENSION_ID\"]}"
     ```

5. Start the stack:

```bash
pnpm docker:up
pnpm dev:stack
```

Ensure an AI provider is configured for AI assist features (`AI_API_KEY` or `OPENROUTER_API_KEY`).

## Usage

1. Sign in on `http://localhost:3000` **or** use **Sign in to JobOS** in the extension popup.
2. Open a job posting (LinkedIn, Indeed, etc.).
3. Click the JobOS icon → **Capture page**.
4. Click **Save & get AI assist** for:
   - Resume fit score + tailoring suggestions
   - Skills/topics to focus on
   - Cover letter (copy button)
   - Networking email (copy button)
   - Suggested interview questions
5. Optional: **Try auto-fill** on simple forms (Greenhouse/Lever/Workday often block extensions).

Reload job/application tabs after installing or updating the extension.

## Development

```bash
pnpm dev:extension
```

Environment variables are loaded from `apps/browser-extension/.env` (generated by `pnpm env:setup`).
